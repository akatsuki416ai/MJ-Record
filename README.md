# MJ 麻將記帳網頁 App

台灣麻將 16 張計台的即時記帳工具。單一記帳者用手機在牌桌旁邊即時記錄每一局結果，App 自動處理莊家連莊與算錢，牌局結束後可把結算摘要上傳到自訂的 webhook（例如接到 Google Sheet）。

## 部署方式

純靜態檔案，沒有建置流程、沒有後端。把整個 `MJ` 資料夾複製到任何靜態網頁主機（Nginx、Apache、Caddy、GitHub Pages 等等皆可）就能用。

**注意**：不能直接雙擊 `index.html` 用 `file://` 開啟——程式用了 ES module（`<script type="module">`），Chrome/Edge 等瀏覽器會在 `file://` 協定下擋掉模組載入（CORS 限制），畫面會整個空白。本機測試請跑一個簡單的靜態伺服器，例如：

```
python -m http.server 8000
```

再用瀏覽器開 `http://localhost:8000`。正式部署到任何 http(s) 主機就完全不會有這個問題。

## 設定

第一次使用請到「設定」畫面：

- **玩家名單**：常駐清單（不含「我」），可以新增/刪除，跨牌局保留，開新牌局時從這裡選人。新增/刪除立即存檔，不用另外按儲存。
- **牌局設定**：座位依逆時針排——「我」（固定是玩家陣列第 0 位，名字存成全域的常駐設定）→ 下家 → 對家 → 上家 → 回到我，跟現實牌桌的莊家輪替方向一致，不用另外設定輪替規則。開始新牌局前，下家/對家/上家要從玩家名單裡選滿 3 個不同的人，「我」的名字不能空白。
- **計分設定**（全域，隨時可編輯）：底 / 每台單價（記錄畫面直接輸入台數，不需要另外設定台數清單）；東（場地費）：每次金額 + 全場收幾次自摸（設 0 代表不收東），規則是全場累計自摸次數中第 1~N 次自摸的贏家除了正常贏錢還要額外付這筆場地費（付給場外，不分給其他三家），超過 N 次之後（「東滿」）就不用再付。這些都是全域預設值，中途調整只會影響之後新輸入的局，不會動到已經記錄的歷史局。
- Webhook 網址（牌局結束上傳用）與傳送模式：
  - `cors`：一般模式，可以讀取上傳是否成功
  - `no-cors`：Google Apps Script 的 Web App 常見情況，瀏覽器讀不到回應內容，畫面只會顯示「已送出，未確認」
- 清除確認碼：要跟 `google-apps-script.gs` 裡的 `CLEAR_CONFIRM_CODE` 常數設一樣的值，結算畫面的「清除 MJ 牌局紀錄」才會生效。

## 記錄畫面

畫面中間是一張模擬牌桌：東南西北排 4 位玩家，「我」固定在南方（下方），下家在東、對家在北、上家在西，跟現實牌桌的莊家輪替方向一致。每位玩家頭像旁邊即時顯示目前損益、自摸/胡牌/被自摸/放槍次數，目前莊家的頭像上會有「莊連N」標記。

- **點某位玩家的頭像**：跳出底部選單，選「自摸」或「胡牌」記錄這一局是誰贏的。
- **選「胡牌」之後**：畫面上方出現「請點選放槍者」提示，直接點另一位玩家頭像就是指定放槍者，不用再跳第二層選單。
- **輸入台數**：贏家（與放槍者）決定後，底部滑出台數輸入面板，輸入台數（不含莊家連莊加成，那部分自動算），確認送出即完成這一局。過程中任何一步都可以按「取消」整個重來。
- **流局**：點桌子正中央的「流局」按鈕即可，莊家連莊不受影響。

## Webhook Payload 格式

有兩種請求，都是 POST 到設定裡填的 webhook 網址，用 `action` 欄位區分：

**牌局結束上傳**（只有最終結算，沒有逐局明細）：

```json
{
  "action": "upload",
  "sessionId": "1752...",
  "startedAt": "2026-07-17T13:00:00.000Z",
  "endedAt": "2026-07-17T17:32:10.000Z",
  "roundCount": 28,
  "settings": { "base": 30, "perTai": 10 },
  "players": [
    {
      "name": "小明",
      "netResult": 320,
      "dongPaid": 150,
      "stats": { "zimo": 3, "hujiao": 5, "beiZimo": 4, "fangChong": 2, "liuju": 6, "meiShi": 8 }
    }
  ],
  "me": { "name": "瓜瓜", "netResult": 100, "dongPaid": 0, "stats": { "...": "..." } }
}
```

`dongPaid` 是這個人這場總共繳了多少東（場地費），`netResult` 已經包含這筆扣款。`stats` 六個欄位對應：自摸次數、胡牌（食胡）次數、被自摸次數、放槍次數、流局次數、沒事次數（食胡局裡既不是贏家也不是放槍者，可以用來算自己的「閃避率」）。`me` 是 `players[]` 裡「我」那一筆的複本，明講出來讓接收端不用去猜陣列順序——寫進「MJ牌局紀錄」分頁用 `players`，寫進「我的紀錄」分頁用 `me`。

**清除牌局紀錄**：

```json
{ "action": "clear", "confirmCode": "跟 Apps Script 裡 CLEAR_CONFIRM_CODE 一樣的值" }
```

只會清空「MJ牌局紀錄」分頁的資料列（保留表頭），「我的紀錄」分頁不受影響、永遠只會累加不會被清除。確認碼不符就什麼都不會清除。

若接收端是 Google Apps Script，`doPost(e)` 可以用 `JSON.parse(e.postData.contents)` 拿到這包資料，用 `payload.action` 判斷要做哪一種處理。若用 `no-cors` 模式送出，Apps Script 那邊仍會正常收到請求本體，只是瀏覽器這邊讀不到回應（包含確認碼錯誤時也一樣讀不到，只能到 Sheet 上確認）。

專案裡的 `google-apps-script.gs` 就是一份可以直接用的接收端程式碼，部署前記得改 `SHEET_ID` 跟 `CLEAR_CONFIRM_CODE` 這兩個常數，部署步驟見該檔案開頭註解。

## 資料保存

牌局進行中的資料存在瀏覽器 localStorage，重新整理頁面不會遺失。牌局結束後資料會移到「上次完成的牌局」，即使上傳失敗也不會被刪除，可以回到結算畫面重試或用「複製 JSON」手動貼到 Sheet。清除瀏覽器資料/換裝置會遺失歷史（沒有雲端資料庫，這是刻意的設計，見專案討論紀錄）。

## 檔案結構

```
index.html
css/style.css
js/
  main.js       -- 啟動、畫面切換
  state.js      -- 資料結構、deriveState()
  scoring.js    -- computeRoundResult()，核心算錢邏輯
  storage.js    -- localStorage 讀寫
  dom.js        -- DOM 建構小工具
  ui-setup.js   -- 設定畫面
  ui-round.js   -- 記錄畫面
  ui-chart.js   -- 損益走勢圖
  ui-summary.js -- 結算/上傳畫面
vendor/chart.umd.min.js  -- vendor 進來的 Chart.js，執行時不吃 CDN
```
